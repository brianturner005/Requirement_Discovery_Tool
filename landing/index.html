The problem

Legacy modernization efforts commonly stall for the same reasons. Requirements surface mid-project, business logic lives only in people's heads, and there's no durable record connecting a requirement to where it came from.

Problem
Impact
Requirements discovered mid-project
Scope creep, missed deadlines
Business logic undocumented
Accidental regression during rewrite
Knowledge locked in individuals
Risk when SMEs leave or are unavailable
Assumptions never written down
Rework when assumptions prove wrong
Decisions not recorded
Future teams repeat past mistakes

What it does

Every requirement is a structured record — not a line in a spreadsheet. Capture where it came from, who owns it, how confident you are, and what evidence backs it up, then move it through a governed approval workflow.

＃

Auto-numbered requirements

Every requirement gets a sequential ID (REQ-001, REQ-002, ...) with rich metadata: title, description, priority, confidence level, and impact.

🔍

Source tracking

Record exactly how each requirement was discovered — SME interview, legacy code analysis, production observation, incident investigation, and more.

✅

Approval workflow

Requirements move through Draft → Under Review → Approved → In Progress → Completed, with invalid transitions rejected server-side.

📎

Evidence attachments

Attach meeting notes, screenshots, log excerpts, and diagrams directly to a requirement. Files are validated and stored safely.

🔗

Linked requirements

Connect related requirements bidirectionally for impact analysis — see what else is affected before you change something.

🏷️

Stakeholders, systems & tags

Maintain reusable directories of people and systems, and tag requirements by domain, risk, or modernization phase.

📊

Live dashboard

See total requirements, status breakdowns, priority distribution, and recent activity at a glance.

🔎

Search & filter

Full-text search across title, description, and ID, plus filtering by status, priority, source, system, and stakeholder.

How it's built

A focused, self-hostable stack — no external services required to run it. SQLite means zero database setup; the React frontend talks to the FastAPI backend over a simple REST API.

v2: Authentication & CSV Export

The latest release adds team accounts with role-based access, plus one-click reporting.

Admin first user

  • Full CRUD on all content
  • Manage user accounts
  • Promote, demote, or deactivate users
  • The first person to register is automatically Admin

Analyst default role

  • Full CRUD on requirements, stakeholders, systems
  • Upload and manage evidence
  • Cannot manage other users
  • Self-registration — no invite required

📥 CSV Export

  • Download the current requirements list as CSV
